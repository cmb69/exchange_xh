Exchange_XH – Handbuch

Exchange_XH ermöglicht den Export und Import der Inhalte einer CMSimple_XH Website nach beziehungsweise von XML. Dies kann nützlich für die Migration zu und von anderen CMSen sein, und ebenfalls für die Migration zwischen CMSimple_XH Versionen.

Wichtiger Hinweis

Exchange_XH konnte bisher nicht in diversen Umgebungen getestet werden. Es ist ein so genannter Release-Candidate. Vorsicht bei der Verwendung!

Voraussetzungen

Exchange_XH ist ein Plugin für CMSimple_XH ≥ 1.6. Es bebötigt PHP ≥ 5.3.0 mit der DOM und SimpleXML Extension.

Installation

Die Installation erfolgt wie bei vielen anderen CMSimple_XH-Plugins auch. Im CMSimple_XH Wiki finden sie ausführliche Hinweise.

  1. Sichern Sie die Daten auf Ihrem Server.
  2. Entpacken Sie die ZIP-Datei auf Ihrem Computer.
  3. Laden Sie das gesamte Verzeichnis exchange/ auf Ihren Server in das plugins/ Verzeichnis von CMSimple_XH hoch.
  4. Vergeben Sie Schreibrechte für die Unterverzeichnisse css/ und languages/.
  5. Navigieren Sie zu Plugins→Exchange im Administrationsbereich, und prüfen Sie, ob alle Voraussetzungen für den Betrieb erfüllt sind.

Einstellungen

Die Konfiguration des Plugins erfolgt wie bei vielen anderen CMSimple_XH-Plugins auch im Administrationsbereich der Homepage. Wählen Sie unter Plugins → Exchange aus.

Die Lokalisierung wird unter Sprache vorgenommen. Sie können die Zeichenketten in Ihre eigene Sprache übersetzen, falls keine entsprechende Sprachdatei zur Verfügung steht, oder sie entsprechend Ihren Anforderungen anpassen.

Das Aussehen von Exchange_XH kann unter Stylesheet angepasst werden.

Verwendung

Sie können den Inhalt der aktuellen Sprache in der Hauptadministration des Plugin exportieren und importieren (Plugins→Import/Export). Nach erfolgreichem Export befindet sich die Datei content.xml im Content-Ordner der aktuellen Sprache, gleich neben der content.htm Datei, aus der exportiert wurde. Um fremde Inhalte zu importieren, müssen Sie eine Datei content.xml, die ein entsprechendes Format haben muss, gleich neben diejenige content.htm Datei platzieren, die Sie durch den Import überschreiben lassen möchten (also legen Sie zuvor ein Backup an).

Bei mehrsprachigen Websites muss der Inhalt jeder Sprache separat importiert und exportiert werden.

Um zwischen CMSimple_XH Versionen zu migrieren, installieren Sie das Plugin zunächst in der alten Version, und exportieren Sie die Inhalte aller Sprachen. Dann muss das Plugin in der neuen Version installiert werden, die zuvor exportierten Inhalte der Sprachen müssen an die passenden Stellen der neuen Version kopiert werden, und alle Sprach-Inhalte müssen importiert werden.

Beschränkungen

Zusätzliches Markup der Seitenüberschriften wird beim Export stillschweigend entfernt.
